### Week 5 Objectives:

- Implement comprehensive security controls (IAM, KMS, WAF, Secrets).
- Enhance visibility with auditing tools (CloudTrail, Config, GuardDuty).

### Tasks to be carried out this week:

| Day | Task                                                                                                                  | Reference        |
| --- | --------------------------------------------------------------------------------------------------------------------- | ---------------- |
| 2   | - **IAM & Secrets:** Review IAM Roles (Least Privilege).<br>- Use Secrets Manager/Parameter Store for credentials.    | AWS IAM/Secrets  |
| 3   | - **Encryption:** Create Customer Managed Keys (CMK) in KMS.<br>- Encrypt S3/EBS with KMS keys.                       | AWS KMS Docs     |
| 4   | - **Edge Security:** Deploy AWS WAF (Web Application Firewall) for ALB/CloudFront.<br>- Test SQLi/XSS blocking rules. | AWS WAF Docs     |
| 5   | - **Compliance:** Enable AWS Config and GuardDuty.<br>- Check Security Hub findings.                                  | AWS Security Hub |
| 6   | - **Review:** Conduct security audit and remediation.<br>- Weekly Report.                                             | -                |

### Week 5 Achievements:

- Centralized secret management using Secrets Manager/SSM.
- Protected endpoints with AWS WAF rules against common attacks.
- Enabled continuous compliance monitoring with AWS Config.
